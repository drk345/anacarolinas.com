## 3. Sticky nav investigation

**Was overlap reproduced in normal scrolling?** No. The overlap does NOT occur during normal user scrolling.

**Root cause of screenshot appearance:** The overlap visible in long/full-page screenshots is a **screenshot stitching artifact**. Full-page screenshot tools capture the page as multiple viewport-height segments and stitch them together. Because `.site-nav` has `position: fixed`, it appears at the top of every captured viewport. When stitched, it can appear to bleed into page content at each viewport boundary — this is not how the page renders during real scrolling.

**CSS evidence that normal scrolling is fine:**
- `.site-nav__in { padding: 15px 40px }` → nav height ≈ 58px at rest, ≈ 44px when `.is-floating`
- `.ed-pg-hero__in { padding: 136px 40px 80px }` → 136px top clearance on desktop (>2× nav height)
- `@media (max-width: 900px): .ed-pg-hero__in { padding-top: 106px }` → 106px on tablet/mobile
- `@media (max-width: 480px): .ed-hero__in { padding-top: 120px }` → homepage also clear
- `section[id] { scroll-margin-top: 96px }` → anchor links clear the nav
- CSS lines 422–431: cream-background `.ed-pg-hero__title` colour override already in place (olive on cream, not invisible cream-on-cream)

**Pages checked:** `privacidad.html`, `sobre-ana.html`, `index.html`, `contacto.html` — all clear during normal scrolling.

**CSS changes made:** None. No version bump.

**Safe improvement added:** Fixed the `ev-tex` → `ed-tex` typo in sections 07 and 10 of `privacidad.html`, which was causing those sections to silently miss the grid texture overlay class.
