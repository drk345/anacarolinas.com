## 2. What works

### Split hero with portrait
The current `programas.html` hero is a flat terracotta (`ed-sec--accent`) text block. The new direction adds a portrait photo on the right half — exactly the pattern the homepage already uses (`ed-hero__in` with `grid-template-columns: 1fr 1fr`). This is a strong improvement: it puts Ana's face on a high-traffic inner page and signals that this is a personal service, not an abstract programme catalogue.

### Cream/white background hero
Moving from terracotta to cream for the inner-page hero is a valid direction. The `sobre-ana.html` hero already uses a cream background with a portrait (`ed-sobre-hero`). Bringing Programas into the same visual register creates coherence between the two most content-rich inner pages.

### Large serif title
The bold, oversized title "ENCUENTRA EL CAMINO QUE ACOMPAÑA TU MOMENTO" reads well at this scale and communicates emotional orientation — not a service catalogue but a personal invitation. The phrasing is strong.

### Outlined CTA button style
The outlined button (`.ed-btn--outline` equivalent) in a cream hero looks cleaner than the filled cream button on a terracotta background. It's already in the editorial system.

### Image selection
The waterfront photo of Ana (appears to be `home-cursos-800.webp` or similar from existing assets) is relaxed and approachable — contrasts well with the seriousness of the method content lower on the page.

## 3. What must be corrected

### Nav label regression — HIGH PRIORITY
**Screenshot shows:** `Sesiones 1:1`
**Must be:** `Sesiones individuales`

This is a routing/SEO regression. The live page is `/sesiones-individuales.html` and every other page in the site calls it "Sesiones individuales" or "Sesiones Individuales". "Sesiones 1:1" does not match the canonical label and could confuse returning visitors.

### "Sob" truncation — LAYOUT BUG
**Screenshot shows:** `Sob` (nav item cut off)
**Must be:** `Sobre mí`

This is a viewport overflow bug in the screenshot viewport, not a deliberate design choice. At the captured width, the nav is running out of space. At wider viewports this likely reads correctly. However it flags a risk: if this design has a wider hero title area, the nav may be tighter. Implementation must validate at ~1280px and ~900px breakpoints.

### CTA wording mismatch
**Screenshot shows:** `AGENDAR CONVERSACIÓN`
**Current standard:** `Agendar conversación inicial`

The word "inicial" is deliberate — it signals a low-commitment first step. All other CTAs across the site (programas, sobre-ana, sesiones, etc.) include "inicial". This must be preserved or explicitly changed by Ana.

### "RESERVAR" in nav — confirm only
The nav CTA button says "Reservar" in the screenshot. Current `programas.html` already uses "Reservar" in the nav CTA (`.ed-navcta`). This is consistent — no change needed.

### ALL CAPS serif title — confirm before adopting
The existing system uses normal-case serif titles (`ed-h2`, `ed-pg-hero__title`, etc.). The ALL CAPS style requires either:
- `text-transform: uppercase` added to a new class
- Or an explicit decision that this page breaks the typographic system

If adopted, it should be a named class (e.g. `.ed-hero-title--caps`) so it can be applied intentionally elsewhere and reverted cleanly. It must NOT be implemented as `style="text-transform:uppercase"` (inline style violates CSP `style-src 'self'`).

### No Claude Design export in repo
No export file was found in the repository. The screenshot appears to be generated by Claude's UI design tool (or Claude Artifacts), not a saved HTML/CSS export. This is good — it means there are no unsafe artifacts to clean up. The implementation must be written from scratch using the editorial CSS system.

### CSP risk — only relevant if export is later provided
If Ana exports HTML from Claude Design, it will almost certainly contain:
- `style=""` inline attributes (blocked by `style-src 'self'`)
- `<style>` blocks inside `<body>`
- External Google Fonts CDN links
- Claude Design runtime scripts (`support.js`, `x-dc` attributes, `DCLogic`)
- `{{ }}` template placeholders

**None of these must be imported into production.** If an export is later provided, treat it as a visual reference only and rebuild using `editorial.css` classes.

## 4. Safe implementation recommendation

**Recommendation: C — Use hero direction + adapt rest of Programas page**

Specifically:
1. Replace the current `ed-sec--accent` hero with a split cream hero matching the new direction
2. Keep all existing content sections below the hero unchanged
3. Do NOT import Claude Design export — build using `editorial.css` classes
4. Confirm CTA wording with Ana before implementing

**Why not D (full redesign)?** The content sections (cards grid, decision helper, comparison table, FAQ, therapy note) are already strong and well-structured. Replacing them purely for visual variety adds risk without user benefit. The hero is where the page falls short visually.

**Why not E (extend to whole site)?** The homepage hero uses a dark olive/terracotta background — very different from this direction. The cream split-hero pattern already exists on `sobre-ana.html`. The correct scope is inner programme pages (`programas.html` first, then evaluate `sesiones-individuales.html`, `conecta.html`, `foco.html`, `intensivo.html`).
